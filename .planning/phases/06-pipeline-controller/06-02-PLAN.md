Create `src/pipeline/spec-compliance.ts`:

1. `checkConvergence(gapHistory: number[]): { converging: boolean; reason: string }` (pure function)
   - Takes the array of gap counts per round: [totalReqs, round1Gaps, round2Gaps, ...]
   - Returns converging=true if the latest count is strictly less than the previous count
   - Returns converging=false with reason if gaps >= previous round
   - Edge case: single entry (first round) is always converging (comparing against baseline)
   - Requirement: PIPE-08

2. `verifyRequirement(requirementId: string, ctx: PipelineContext): Promise<{ passed: boolean; gapDescription: string }>`
   - Runs a step via `runStep()` that asks the agent to verify whether a specific requirement is met
   - Uses `outputSchema` for structured output: `{ passed: boolean, gapDescription: string }`
   - The prompt includes the requirement ID and asks the agent to check the codebase
   - Returns the structured result
   - Requirement: PIPE-07

3. `runSpecComplianceLoop(requirementIds: string[], ctx: PipelineContext): Promise<SpecComplianceResult>`
   - Implements the loop from SPEC.md lines 534-598:
     a. Seed gapHistory[0] = requirementIds.length (baseline)
     b. For each round up to `config.maxComplianceRounds`:
        - Verify each requirement via `verifyRequirement()`
        - Collect gaps (requirements that didn't pass)
        - Update state.specCompliance: gapHistory[round] = gaps.length, verified = total - gaps, roundsCompleted = round
        - If gaps.length === 0: converged! Return { converged: true, ... }
        - Check convergence: if gaps >= previous round, stop (not converging)
        - Fix each gap via `runStep()` with `buildComplianceGapPrompt()`
     c. If exhausted maxRounds: return { converged: false, remainingGaps: [...] }
   - Updates `state.status` to "wave_3" at start, updates `state.remainingGaps` if not converged
   - Requirements: PIPE-07, PIPE-08
