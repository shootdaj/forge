  3. Final result is { status: "completed" }
  4. State tracks full gapHistory

- `TestPipelineScenario_FullRun_WithSkippedItems` -- Wave 1 produces a skipped item. Checkpoint includes it. Resume with guidance. Wave 2 addresses skipped item using guidance. Verify guidance text appears in fix prompt.

- `TestPipelineScenario_ComplianceNotConverging` -- Wave 3 spec compliance loop does not converge (gaps stay the same). Result: { status: "stuck" }. State: status="stuck", remainingGaps populated, specCompliance.roundsCompleted > 0.

- `TestPipelineScenario_PhaseFails_ContinuesOthers` -- one phase fails during Wave 1, other independent phases still execute. Skipped items include the failed phase's requirements. Verify pipeline doesn't halt on first failure.
