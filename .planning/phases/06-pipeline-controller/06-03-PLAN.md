Create `src/pipeline/pipeline-controller.ts` implementing `runPipeline()`:

The function signature:
```typescript
export async function runPipeline(ctx: PipelineContext): Promise<PipelineResult>
```

The FSM follows SPEC.md section 3 (Pipeline Controller), with state transitions driven by code:

**Phase 0: Initialization**
- Load state via `ctx.stateManager.load()`
- If `state.status === "completed"` or `state.status === "failed"`, return immediately
- If `state.status === "human_checkpoint"`, skip directly to resume logic (Wave 2)
- Read roadmap via `ctx.fs.readFileSync` (path from state or default `.planning/ROADMAP.md`)
- Parse phases and build execution waves via `getExecutionWaves(roadmapContent)`
- Collect all requirement IDs across all phases

**Wave 1: Build everything with mocks** (PIPE-01)
- Update state.status = "wave_1", state.currentWave = 1
- Create a `MockManager` instance
- For each execution wave from topological sort:
  - For each phase in the wave (sequential for v1, graph supports concurrent):
    - Skip if `state.phases[N].status === "completed"`
    - Detect external services via `mockManager.detectExternalServices(phase.description, phase.number)`
    - Build mock instructions via `mockManager.buildMockInstructions(detectedServices)`
    - Call `ctx.runPhaseFn(phase.number, phaseRunnerCtx, { mockInstructions, requirementIds: phase.requirementIds })`
    - Register detected mocks via `mockManager.registerMock()` for each service
    - Collect services needed and skipped items from result
    - Update `state.servicesNeeded` and `state.skippedItems` after each phase
    - Save state after each phase

**Human Checkpoint** (PIPE-04)
- After Wave 1 completes, check `needsHumanCheckpoint(state)`
- If checkpoint needed:
  - Update state.status = "human_checkpoint"
  - Generate checkpoint report via `generateCheckpointReport(state)`
  - Write checkpoint file via `writeCheckpointFile(report, "forge-checkpoint.json")`
  - Format and log display via `formatCheckpointDisplay(report)`
  - Return `{ status: "checkpoint", report }` -- pipeline pauses here
- If no checkpoint needed, continue to Wave 2

**Wave 2: Real integrations** (PIPE-05, PIPE-06)
- Update state.status = "wave_2", state.currentWave = 2
- If `state.servicesNeeded.length > 0`:
  - Get mock registry via `mockManager.getMockRegistry()`
  - Build integration prompt via `buildIntegrationPrompt(state.servicesNeeded, state.credentials)`
  - Also build swap prompt via `mockManager.buildSwapPrompt(registry, state.credentials)`
  - Call `runStep("integrate-real-services", { prompt: combinedPrompt, verify: ... })` via step runner
- If `state.skippedItems.length > 0` (PIPE-06):
  - For each skipped item:
    - Get guidance from `state.humanGuidance[item.requirement]` (or empty string)
    - Build prompt via `buildSkippedItemPrompt(item, guidance)`
    - Call `runStep("fix-skipped-${item.requirement}", { prompt, verify: ... })`

**Wave 3+: Spec compliance** (delegates to runSpecComplianceLoop)
- Update state.status = "wave_3", state.currentWave = 3
- Call `runSpecComplianceLoop(allRequirementIds, ctx)`
- If not converged: update state.status = "stuck", return `{ status: "stuck", remainingGaps, roundsCompleted }`

**UAT gate** (PIPE-09)
- After spec compliance converges:
  - Update state.status = "uat"
  - Call `runStep("run-uat", { prompt: "Run user acceptance testing...", verify: ... })`
  - If UAT fails, attempt gap closure + retry (up to config.maxRetries)
  - If UAT still fails: return `{ status: "failed", reason: "UAT failed", wave: 4 }`

**Milestone completion** (PIPE-10)
- Call `runStep("audit-milestone", { prompt: "Run /gsd:audit-milestone...", verify: ... })`
- If audit finds gaps, call `runStep("plan-milestone-gaps", ...)` and run gap phases
- Call `runStep("complete-milestone", { prompt: "Run /gsd:complete-milestone...", verify: ... })`
- Update state.status = "completed"
- Return `{ status: "completed", summary: "..." }`

**Error handling:**
- Wrap each wave in try/catch
- On error: update state.status = "failed", save state, return `{ status: "failed", reason, wave }`
- Budget exceeded errors bubble up from runStep -- caught and reported

**Helper: buildPhaseRunnerCtx(ctx: PipelineContext): PhaseRunnerContext**
- Converts PipelineContext to PhaseRunnerContext (they share most fields)
- Used when calling runPhaseFn

Important: This function is long but linear. Each wave is a clearly separated section with state transitions at boundaries. No complex branching -- just sequential wave execution with early returns on failure.
