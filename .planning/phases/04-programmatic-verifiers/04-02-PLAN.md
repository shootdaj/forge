  <action>
    **1. Create `src/verifiers/index.ts`:**

    **Registry map:**
    ```typescript
    const verifierRegistry: Record<string, Verifier> = {
      files: filesVerifier,
      tests: testsVerifier,
      typecheck: typecheckVerifier,
      lint: lintVerifier,
      coverage: coverageVerifier,
      observability: observabilityVerifier,
      docker: dockerVerifier,
      deployment: deploymentVerifier,
    };
    ```

    **Config-to-registry name mapping:**
    Map ForgeConfig.verification property names to registry keys:
    - `files` -> `"files"`
    - `tests` -> `"tests"`
    - `typecheck` -> `"typecheck"`
    - `lint` -> `"lint"`
    - `testCoverageCheck` -> `"coverage"`
    - `observabilityCheck` -> `"observability"`
    - `dockerSmoke` -> `"docker"`
    - `deployment` -> `"deployment"`

    **`getEnabledVerifiers(forgeConfig: ForgeConfig): string[]`:**
    - Read the verification config object
    - Return array of registry names where the toggle is true
    - Use the mapping above to translate camelCase config keys to registry names

    **`runVerifiers(config: VerifierConfig): Promise<VerificationReport>`:**
    - Record start time with `Date.now()`
    - Call `getEnabledVerifiers(config.forgeConfig)` to get enabled list
    - Separate into `nonDocker` (all except "docker") and `dockerEnabled` (boolean)
    - **Phase 1 — parallel non-docker:**
      - Run `Promise.allSettled(nonDocker.map(name => verifierRegistry[name](config)))`
      - For each settled result:
        - `fulfilled`: use the VerifierResult directly
        - `rejected`: create a synthetic failed result: `{ passed: false, verifier: name, details: [], errors: ["Verifier threw: " + error.message] }`
    - Determine `allNonDockerPassed`: every non-docker result has `passed: true`
    - **Phase 2 — conditional docker:**
      - If `dockerEnabled && allNonDockerPassed`: run `verifierRegistry.docker(config)` and add result
      - If `dockerEnabled && !allNonDockerPassed`: add skipped result: `skippedResult("docker", "Skipped: non-docker verifiers failed")`
    - **Aggregate report:**
      - `passed`: every result in the final array has `passed: true`
      - `results`: the full array of VerifierResult
      - `summary.total`: results.length
      - `summary.passed`: results where passed: true AND not skipped (details[0] doesn't start with "Skipped:")
      - `summary.failed`: results where passed: false
      - `summary.skipped`: results where details[0] starts with "Skipped:" (the skippedResult pattern)
      - `durationMs`: Date.now() - startTime

    **Re-exports:**
    - Export all types from `./types.js`
    - Export `runVerifiers`, `getEnabledVerifiers`, `verifierRegistry`
    - Export individual verifiers for direct use
  </action>

    - VerificationReport aggregated with correct summary counts
