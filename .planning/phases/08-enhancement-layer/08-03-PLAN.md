Create `src/uat/workflows.ts` with:

1. `extractUserWorkflows(requirementsContent: string, appType: AppType): UATWorkflow[]` — Parses REQUIREMENTS.md (the formatted output from the requirements module). For each `## R{N}: Title` section, extracts acceptance criteria bullet points and converts each into a UAT workflow. The workflow ID is `UAT-R{N}-{index}`. Each workflow's steps come from the acceptance criteria. The appType is applied to all workflows (detected from config by the caller).

2. `buildSafetyPrompt(safetyConfig: SafetyConfig): string` — Builds the safety guardrail portion of UAT prompts. Lists constraints: "NEVER use production credentials", "Use .env.test for all environment variables", "Use local SMTP (Mailhog/Mailtrap) for email testing", "Use test database from Docker container", "Use test OAuth apps / mock providers only". Returns the prompt string.

3. `runUATGapClosure(failedWorkflows: WorkflowResult[], ctx: UATContext): Promise<void>` — For each failed workflow, calls runStep() with a prompt that includes the workflow ID, the failure errors, and instructions to create a targeted fix. Each fix step is named `uat-fix-{workflowId}`. Uses ctx.stepRunnerContext and ctx.costController. Does not throw -- gap closure failures are logged but not fatal.

Create `src/uat/runner.ts` with:

1. `detectAppType(config: ForgeConfig): AppType` — Inspects config.testing.stack. Returns "web" for stacks like "react", "next", "vue", "angular", "svelte". Returns "api" for "express", "fastify", "nestjs", "django", "flask", "rails". Returns "cli" for "node", "python", "go", "rust" and any other. Default: "cli".

2. `startApplication(config: ForgeConfig, ctx: UATContext): Promise<boolean>` — Checks if docker-compose file exists (using ctx.fs or real fs). If exists, runs `docker compose -f {file} up -d` via ctx.execFn or real execSync. Returns true on success, false on failure (no throw). If no docker-compose file, returns true (assume app can be tested directly).

3. `stopApplication(config: ForgeConfig, ctx: UATContext): Promise<void>` — Runs `docker compose -f {file} down` if docker-compose file exists. Catches errors silently.

4. `waitForHealth(healthUrl: string, timeoutMs: number, ctx: UATContext): Promise<boolean>` — Polls the health URL every 2 seconds up to timeoutMs. Uses ctx.execFn to run `curl -sf {url}` and checks exit code. Returns true when healthy, false on timeout. Default healthUrl: "http://localhost:3000/health". Default timeoutMs: 30000.

5. `buildUATPrompt(workflow: UATWorkflow, appType: AppType, safetyPrompt: string): string` — Builds the test prompt for a single workflow. For "web": instructs agent to use headless browser (Playwright via MCP or agent-browser CLI). For "api": instructs agent to use curl/fetch via bash. For "cli": instructs agent to run commands and check stdout/stderr/exit codes. Includes the workflow steps as numbered items. Appends safety prompt. Instructs agent to write JSON results to `.forge/uat/{workflowId}.json` with format `{ passed: boolean, stepsPassed: number, stepsFailed: number, errors: string[] }`.

6. `verifyUATResults(workflowId: string, forgeDir: string, ctx: UATContext): WorkflowResult` — Reads `.forge/uat/{workflowId}.json` from filesystem (using ctx.fs or real fs). Parses JSON. Returns WorkflowResult. If file doesn't exist or is invalid JSON, returns a failed result with appropriate error.

7. `runUAT(ctx: UATContext): Promise<UATResult>` — Main entry point. Flow:
   a. Read REQUIREMENTS.md via ctx.fs
   b. Detect app type via detectAppType(config)
   c. Extract workflows via extractUserWorkflows(content, appType)
   d. Build safety config and prompt via buildSafetyPrompt()
   e. Start application via startApplication()
   f. Wait for health via waitForHealth()
   g. For each workflow: call runStep() with buildUATPrompt(), then verifyUATResults()
   h. Stop application via stopApplication()
   i. Aggregate results into UATResult
   j. If any failed AND attempts < maxRetries: call runUATGapClosure(), then retry the failed workflows
   k. Update state with final UAT results
   l. Return UATResult with status "passed" if all pass, "failed" if some fail, "stuck" if gap closure doesn't help after max retries
