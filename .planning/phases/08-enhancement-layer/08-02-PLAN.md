```typescript
export interface NotionPageIds {
  architecture: string;
  dataFlow: string;
  apiReference: string;
  componentIndex: string;
  adrs: string;
  deployment: string;
  devWorkflow: string;
  phaseReports: string;
}

// The 8 mandatory page names (used for creation prompts)
export const MANDATORY_PAGES = [
  "Architecture",
  "Data Flow",
  "API Reference",
  "Component Index",
  "ADRs",
  "Deployment",
  "Dev Workflow",
  "Phase Reports",
] as const;

export interface PhaseReport {
  phaseNumber: number;
  phaseName: string;
  goals: string;
  testResults: { passed: number; failed: number; total: number };
  architectureChanges: string[];
  issues: string[];
  budgetUsed: number;
}

export interface ADRRecord {
  title: string;
  context: string;
  decision: string;
  consequences: string;
  status: "accepted" | "superseded" | "deprecated";
}

export interface MilestoneSummary {
  totalPhases: number;
  totalTests: number;
  totalBudget: number;
  requirements: { total: number; verified: number };
  highlights: string[];
}

/** Injectable executeQuery function type for testing */
export type ExecuteQueryFn = (options: {
  prompt: string;
  model?: string;
  [key: string]: unknown;
}) => Promise<{ ok: boolean; result?: string; error?: { message: string } }>;
```

Create `src/docs/notion.ts` with these functions. All functions accept an injectable `executeQueryFn` parameter for testability (same pattern as other modules). Each function builds a prompt for the agent that uses Notion MCP tools.

1. `createDocPages(parentPageId: string, projectName: string, options?: { executeQueryFn?: ExecuteQueryFn, model?: string }): Promise<NotionPageIds>` — Builds a prompt instructing the agent to create 8 child pages under the parent page ID using the Notion MCP `notion_create_page` tool. The prompt lists all 8 page names and asks the agent to return the created page IDs as structured JSON. Uses outputSchema to extract `{ architecture: string, dataFlow: string, ... }`. On SDK failure, throws descriptive error. Returns NotionPageIds.

2. `buildPageUpdatePrompt(pageId: string, pageName: string, content: string): string` — Pure function that builds a prompt for updating a specific Notion page. The prompt instructs the agent to read the current page content via `notion_read_page`, then apply the provided content as an update via `notion_update_page`. Returns the prompt string.

3. `updateArchitecture(pageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Calls executeQueryFn with a prompt to update architecture page with data from phase report. Extracts architecture changes and formats them.

4. `updateDataFlow(pageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Same pattern, updates data flow page.

5. `updateApiReference(pageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Same pattern.

6. `updateComponentIndex(pageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Same pattern.

7. `updateDevWorkflow(pageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Same pattern.

8. `createADR(parentPageId: string, adr: ADRRecord, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<string>` — Creates a new ADR page under the ADRs parent. Returns created page ID.

9. `createPhaseReport(parentPageId: string, report: PhaseReport, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<string>` — Creates a phase report page under Phase Reports parent. Content includes: goals, test results table, architecture changes list, issues list, budget spent. Returns created page ID.

10. `publishFinalDocs(pageIds: NotionPageIds, summary: MilestoneSummary, options?: { executeQueryFn?: ExecuteQueryFn }): Promise<void>` — Updates each page with "Final - Milestone Complete" status. Creates a milestone completion summary page under phaseReports.

All update functions follow the same pattern: build prompt with buildPageUpdatePrompt, call executeQueryFn, handle errors gracefully (log warning but don't throw -- Notion updates are non-critical).
