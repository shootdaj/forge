Create `src/requirements/types.ts` with these type definitions:

```typescript
export type RequirementCategory = "Core" | "Data" | "Security" | "Integrations" | "Quality" | "Infrastructure" | "UX" | "Business";

export interface Requirement {
  id: string;           // "R1", "R2", etc.
  title: string;
  category: RequirementCategory;
  description: string;
  acceptanceCriteria: string[];
  edgeCases: string[];
  performance?: string;
  security?: string;
  observability?: string;
}

export interface ComplianceFlags {
  soc2: boolean;
  hipaa: boolean;
  gdpr: boolean;
  pciDss: boolean;
  wcag: boolean;
}

export interface GatherResult {
  requirements: Requirement[];
  complianceFlags: ComplianceFlags;
  rawOutput: string;
  formattedDoc: string;
}
```

Create `src/requirements/parser.ts` with three pure functions:

1. `parseRequirementsOutput(rawOutput: string): Requirement[]` — Parses markdown output with `## R{N}: Title` headers. Extracts fields from bold-labeled lines (**Description:**, **Acceptance Criteria:**, **Edge Cases:**, **Performance:**, **Security:**, **Observability:**). Acceptance criteria and edge cases are parsed as bullet lists (lines starting with `- `). If a section is missing, default to empty string/array. Category is extracted from a **Category:** line if present, defaults to "Core".

2. `detectComplianceFlags(requirements: Requirement[]): ComplianceFlags` — Scans all requirement text (description, acceptance criteria, edge cases, security) for compliance keywords. SOC 2: "soc 2", "soc2", "audit log". HIPAA: "hipaa", "phi", "protected health". GDPR: "gdpr", "data protection", "right to erasure", "consent". PCI DSS: "pci", "payment card", "cardholder". WCAG: "wcag", "accessibility", "a11y", "screen reader". Case-insensitive matching. Returns object with boolean flags.

3. `formatRequirementsDoc(requirements: Requirement[], complianceFlags: ComplianceFlags): string` — Formats REQUIREMENTS.md content. Header with title and date. Compliance section listing active flags. Then each requirement formatted as:
```
## R1: Title
**Category:** ...
**Description:** ...
**Acceptance Criteria:**
- item 1
- item 2
**Edge Cases:**
- edge 1
**Performance:** ...
**Security:** ...
**Observability:** ...
```

Create `src/requirements/gatherer.ts` with two functions:

1. `buildRequirementsPrompt(projectName?: string): string` — Returns a comprehensive prompt string instructing the agent to gather requirements across all 8 categories. The prompt lists 25+ topics organized by category:
   - Core: functionality, user workflows, personas, edge cases, error handling
   - Data: data model, lifecycle, state management, storage, migrations
   - Security: authentication, authorization, compliance, encryption, audit
   - Integrations: third-party services, APIs, webhooks, notifications, payments
   - Quality: performance targets, error handling, resilience, validation, testing
   - Infrastructure: deployment, observability, CI/CD, environments, scaling
   - UX: design patterns, accessibility, i18n, responsive, loading states
   - Business: launch scope, success metrics, acceptance criteria, MVP vs full

   The prompt instructs the agent to output requirements in the `## R{N}: Title` markdown format with all structured fields (Description, Acceptance Criteria as bullet list, Edge Cases as bullet list, Performance, Security, Observability, Category).

2. `gatherRequirements(config: ForgeConfig, options?: { executeQueryFn?: typeof executeQuery, projectName?: string }): Promise<GatherResult>` — Main entry point. Builds the prompt via buildRequirementsPrompt(). Calls executeQueryFn (defaulting to executeQuery from SDK) with the prompt, the config model, useClaudeCodePreset: true, loadSettings: true. On success, pipes result.result through parseRequirementsOutput(), detectComplianceFlags(), and formatRequirementsDoc(). Returns GatherResult. On failure, throws with descriptive error message.

- parseRequirementsOutput, detectComplianceFlags, formatRequirementsDoc from parser.ts
