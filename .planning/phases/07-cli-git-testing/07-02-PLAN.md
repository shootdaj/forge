```typescript
export function createCli(): Command {
  const program = new Command();
  program.name('forge').description('Autonomous software development orchestrator').version('0.1.0');

  // forge init
  program.command('init').description('Start interactive requirements gathering')
    .action(async () => { ... });

  // forge run
  program.command('run').description('Execute full wave model autonomously')
    .action(async () => { ... });

  // forge phase <number>
  program.command('phase <number>').description('Run a single phase')
    .action(async (phaseNum: string) => { ... });

  // forge status
  program.command('status').description('Display project status')
    .action(async () => { ... });

  // forge resume
  program.command('resume').description('Continue from checkpoint')
    .option('--env <file>', 'Path to .env file with credentials')
    .option('--guidance <file>', 'Path to guidance markdown file')
    .action(async (opts) => { ... });

  return program;
}
```

Command handler details:

**forge init**: Load config via `loadConfig()`. Create initial state via `createInitialState()` and `StateManager`. Log "Requirements gathering will be available in Phase 8. Project initialized." For now this is a stub that creates config and state files. Later Phase 8 will implement the full interactive questionnaire. Still creates the TEST_GUIDE.md (calls `createTestGuide` with empty requirements -- will be populated later) and injects testing methodology (calls `injectTestingMethodology`).

**forge run**: Load config, create/load state manager. Create CostController. Build StepRunnerContext with `executeQuery` as the `executeQueryFn`. Build PipelineContext with `runPhase` as `runPhaseFn`. Call `runPipeline(ctx)`. Handle the PipelineResult discriminated union: print success summary for "completed", print checkpoint report for "checkpoint" (including resume instructions), print error for "failed", print non-convergence for "stuck". Exit with code 0 on completed, 1 on failed/stuck.

**forge phase N**: Parse phase number. Load config/state. Create CostController + StepRunnerContext. Build PhaseRunnerContext. Call `runPhase(parseInt(phaseNum), ctx)`. Handle PhaseResult: print report for "completed", print error for "failed", print partial progress for "partial". Does NOT do wave model -- just runs that single phase.

**forge status**: Load state via StateManager (if state file exists -- if not, print "No forge project found. Run `forge init` first." and exit). Call `formatStatus(state)`. Print to stdout. Also pass `config.maxBudgetTotal` to the budget formatter (load config too, or use default).

**forge resume**: Require `--env` option (error if missing). Load config/state. Call `loadResumeData(envFile, guidancePath)`. Update state with credentials and guidance via `stateManager.update()`. Build PipelineContext. Call `runPipeline(ctx)`. Same result handling as `forge run`.

Helper function `buildPipelineContext(config, stateManager, costController)` to avoid duplication between run/resume.

Error handling: Wrap each command action in try/catch. On error, print `Error: {message}` to stderr and exit with code 1. On `BudgetExceededError`, print a specific budget message.

Add `export async function main() { const cli = createCli(); await cli.parseAsync(process.argv); }` at the bottom.
